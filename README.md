# 回路基板検査装置

## 1. 制作背景

### 概要
香川高専在学時の実験実習（PBL学習）として、株式会社Rexxamとの産学共同プロジェクトで制作しました。

### プロジェクト条件
- 期間： 約3ヶ月
- 予算： 15,000円（購入物品の合計：¥14,466）
- チーム： 4人（同級生チーム）
- 成果発表： 制作物の発表・試運転披露を企業にて実施。企業幹部陣からのフィードバック・質疑応答あり

### 課題・目的
工場の製造ラインへの組み込みを想定した、電子回路基板の素子欠陥・欠損を自動検出する検査装置の開発。

限られた予算・期間の中で、単に動作するものを作るだけでなく、時間・予算・品質のプロジェクト計画立案から遂行まで一貫して行うことが求められました。

- 予算制約（¥15,000）のもと、LEGO部品・アルミフレーム・3Dプリンタパーツを組み合わせたコスト最適設計を実施（実績：¥14,466）
- 発表会では企業幹部陣の前で試運転を披露し、フィードバックを受領

### 自分の担当領域
本プロジェクトにおける自分の主な担当は以下の3領域です。

- 画像解析（全般）： 前処理（明るさ補正・Affine変換・分割撮影結合）から欠陥検出（コンデンサ方向検出・素子欠損検出）まで、IMAGE モジュール全体の設計・実装
- GUI 設計および一部実装： 画面レイアウト設計、場面に応じたダイアログの設計と一部コーディング
- ハードウェアの設計および製作： 暗箱・ベルトコンベア・電装まわりの設計・加工・組み立て

## 2. 制作した成果物

### システム全体構成

システムは以下の4つのモジュールで構成されています。

| モジュール | 担当機能 |
|---|---|
| GUI モジュール | アプリケーション画面・ダイアログの制御、検査結果の表示 |
| IMAGE モジュール | 画像撮影・処理・良品/不良品の判別 |
| MOTOR モジュール | ベルトコンベアのステッピングモーター制御 |
| DATA モジュール | 検査履歴のCSV保存・管理 |

PCとRaspberry Piが連携して動作し、GUIからの操作で一連の検査フローが自動実行されます。

### 検査フロー

1. スタート位置に検査基板をセット
2. GUIを操作し、検査開始
3. 検査対象を複数回に分けて撮像
4. 画像処理で基板の良・不良を判別
5. 良品ならベルトコンベアが順転、不良品なら逆転して返却

### ハードウェア

限られた予算の中で以下を自作・組み立てました。

- 暗箱： レーザー加工機でアクリル板を加工。外光を遮断しつつ、小さい出入口と内部のテープライトで均一な撮影環境を実現。配線は内部に隠し、整然とした設計を維持。
- ベルトコンベア： アルミフレーム・LEGOキャタピラ・3Dプリンタ製パーツを組み合わせて自作。ステッピングモーター2基を同期制御し、基板を精度よく搬送。
- 電装： ブレッドボード配線は信号種類ごとに色分けし、ターミナルブロックにより接続・交換を容易にしています。

### ソフトウェア（画像処理）

#### 画像の前処理
- 分割撮影・結合： 基板を複数回に分けて撮影し画像を結合することで、1回撮影より高画質（最大約2300×720画素）を実現しつつ、暗箱を小型化。外光の影響を受ける画像の上下端は結合時にクリップ。
- 明るさ補正： LEDライトの暖色系の色かぶりを補正し、明るさ・ハイライトを調整。
- Affine変換： 基板の緑色部分を抽出・外接矩形を算出し、OK基板と検査基板のサイズ・位置を正規化。

#### 欠陥検出
- コンデンサの方向検出： Hough変換で円形を抽出。コンデンサ以外（緑など）をフィルタリングした後、検査基板の円形画像と上下左右反転させた円形画像の両方をOK基板と差分比較し、差分が小さい方が正方向と判定。
- 素子の欠損検出： OK基板から白・黒・赤・青の4色マスクを生成し、検査基板との差分から欠損箇所を検出。面積・辺長の閾値でノイズを除外し、欠損部分をハイライト表示。

### GUI

- PyQt / Tkinter 系のGUIアプリケーションとして実装
- 場面に応じたダイアログを実装（検査画面・登録・閲覧画面など）
- 検査履歴はPandasのDataFrameで管理し、タイムスタンプ・結果・欠陥数・欠陥位置をCSVで保存・表示

## 3. 技術面において工夫した点

### ① 機械学習を使わない画像処理アプローチの採用

クライアント（企業）が本質的に求めているものを分析した結果、高精度なCNNモデルよりも、課題に対して工夫されたアルゴリズムの設計力を評価していると判断しました。
そのため、性能という絶対的な価値を捨ててでも工夫したアルゴリズムを実装することに注力しました。この本質的な要求を捉えることができたからこそ、企業での最終発表で非常によい評価を受け取ったのだと感じています。

また、工場ラインで使用するマイクロコンピュータの性能上、機械学習モデルの推論時間がライン速度の要求を満たさないと判断しました。そのため、性能の絶対値を求めるのではなく、従来の画像処理技術（差分処理・Hough変換・色抽出など）を組み合わせた軽量かつ高速なアルゴリズムを独自設計しました。

### ② GUIアプリケーションの開発による利便性の向上

本実習でGUIの実装は必須要件ではありませんでしたが、工場現場でのオペレーター利用を想定し、あえてGUIを設計・実装しました。場面に応じたダイアログや検査履歴の視覚的な表示により、専門知識がなくても直感的に操作できるUXを実現しています。

### ③ 閾値を用いないコンデンサ方向検出アルゴリズムの設計

コンデンサの向き検出において、一般的な閾値との比較ではなく、正方向の差分と反転させた場合の差分を相対比較するアプローチを採用しました（詳しくは `回路基板検査装置.pptx`をご参照ください）。

閾値ベースの手法では照明条件や基板の個体差によってチューニングが必要になりますが、このアプローチでは両差分の大小関係のみで判定するため、環境変化に対して頑健性のある検出が可能です。

## 4. プロジェクト遂行にあたって工夫した点

### ① Teamsを活用したコミュニケーション設計

4人チームで約3ヶ月という短期間のプロジェクトを円滑に進めるため、Microsoft Teamsをメインのコミュニケーション基盤として活用しました。具体的には以下のルールをチームで定めました。
これらのルール制定をしっかりと徹底したことで限られた予算や期限においても満足のいく成果物が作成できたのだと感じています。

- すべてのメッセージ通知をONにして情報の見落としを防止
- 確認済みのメッセージにはリアクションを付け、既読状態を可視化
- 進捗報告を頻繁に行い、タスクの停滞を早期発見
- 積極的に意見を出し合う文化を醸成し、意見の相違によるプロジェクト停滞を回避

### ② タスク分割と並行開発

ハード・ソフトを並行して進めるため、「暗箱の作成」「コンベア部分の作成」「アルミフレームの外枠作成」の3タスクに分割し、役割を明確化して同時進行しました。ソフトウェア側でもGUI・画像処理・モーター制御・データ管理の各モジュールを分離設計し、独立して開発・統合できる構成にしました。
各個人が独立した部分を開発することによって、各個人がある程度の責任を持つことができ、効率のよい有意義な開発を行うことができました。
